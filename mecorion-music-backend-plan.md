# План backend-разработки Mecorion Music

## Цель первого этапа

Создать отдельный TypeScript API, который предоставляет музыкальный каталог и
хранит данные в PostgreSQL. Первая версия должна быть простой для изучения, но
иметь границы, подходящие для дальнейшего роста.

## Этап 1. Основа API

- отдельное приложение `apps/music-api`;
- Fastify как HTTP-сервер;
- Zod для проверки входных данных;
- `pg` для параметризованных SQL-запросов;
- единый формат ошибок;
- проверка состояния API и базы данных;
- CORS для локального frontend-приложения;
- конфигурация только через переменные окружения.

Результат: API запускается независимо от frontend и отвечает на `/health`.

## Этап 2. PostgreSQL

- SQL-миграции с журналом выполненных версий;
- таблицы исполнителей, альбомов и треков;
- жанры и связь треков с жанрами;
- плейлисты и позиции треков;
- избранное и история воспроизведения;
- тексты песен;
- индексы для основных запросов;
- тестовый seed каталога.

Результат: чистую базу можно развернуть одной командой и получить тестовый
каталог.

## Этап 3. Каталог

- получение списка треков;
- фильтрация по исполнителю, альбому, году и доступности;
- пагинация и сортировка;
- получение отдельного трека;
- создание трека для административного сценария;
- списки исполнителей и альбомов;
- OpenAPI-документация.

## Этап 4. Пользовательская медиатека

- интеграция с общей авторизацией Mecorion;
- лайки;
- история прослушивания;
- CRUD плейлистов;
- изменение порядка треков;
- приватность плейлистов;
- синхронизация очереди и настроек.

## Этап 5. Аудиофайлы

- S3-совместимое объектное хранилище;
- загрузка исходного файла;
- фоновая обработка через FFmpeg;
- извлечение метаданных;
- несколько вариантов качества;
- временные подписанные ссылки;
- проверка прав и региональной доступности.

## Этап 6. Поиск и рекомендации

- полнотекстовый поиск через Meilisearch или OpenSearch;
- сбор событий прослушивания;
- агрегация статистики;
- радио по треку и исполнителю;
- персональные подборки;
- кеширование популярных запросов в Redis.

## Этап 7. Эксплуатация

- unit- и integration-тесты;
- отдельная тестовая PostgreSQL;
- Docker-образ API;
- CI с миграциями и тестами;
- структурированные логи;
- метрики и трассировка;
- резервное копирование;
- rate limiting и аудит административных операций.

## Ближайшие следующие задачи

1. Подключить frontend к `GET /api/v1/tracks`.
2. Добавить OpenAPI и Swagger UI.
3. Реализовать обновление и удаление треков.
4. Подключить авторизацию Mecorion и таблицу профилей Music.
5. Реализовать CRUD плейлистов.
6. Добавить integration-тесты маршрутов с отдельной базой.
